# Task 020: Testing Suite & Module Documentation

## Context
Tasks 014–019 delivered the complete pitch detection module with smoothing and edge case handling. This task adds automated tests and documentation so the module is production-ready and maintainable.

## Steps
1. Create C++ unit tests (using a lightweight approach — no heavy test framework needed, simple assert-based harness):
   - YIN accuracy: sine waves at 82.4Hz (E2), 220Hz (A3), 440Hz (A4), 880Hz (A5), 4186Hz (C8). Verify ±1 cent for mid-range, ±5 cents at extremes.
   - YIN with noise: add gaussian noise at SNR=20dB, verify detection still works above confidence threshold.
   - YIN with silence: all-zero buffer returns confidence < 0.5.
   - Ring buffer: test write, read, wrap-around, overflow behavior.
   - Note mapper: verify A4=440Hz → "A", octave 4, 0 cents. Verify edge cases (C4=261.63Hz, sharps/flats).
   - EMA filter: verify smoothing behavior and reset on large jumps.
2. Enhance the test screen at `app/pitch-test.tsx` (created in task 018):
   - Display: frequency, note, octave, cents, confidence, latency.
   - Useful for manual validation with real instruments or virtual instruments.
3. Write a `README.md` for the module covering:
   - Installation (how to add to an Expo app).
   - API reference (`usePitchDetector` hook, options, return values).
   - Architecture overview (one paragraph + diagram).
   - Performance characteristics (latency, accuracy, CPU usage).
   - Known limitations and future work (polyphonic, chord detection).
4. Add inline code comments to the C++ DSP files explaining the YIN algorithm steps for future maintainers.

## Acceptance Criteria
- [ ] C++ test harness passes all frequency accuracy tests
- [ ] Ring buffer tests cover wrap-around and edge cases
- [ ] Test screen in app displays live pitch data (frequency, note, octave, cents, confidence, latency) on both platforms
- [ ] README.md covers installation, API, architecture, and performance
- [ ] YIN algorithm steps have clear inline comments
- [ ] All tests can be run with a single command (C++ harness via shell script or Makefile)
