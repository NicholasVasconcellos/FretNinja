# Task 003: State Management & Persistence

## Context
Tasks 001-002 established the project structure and utility functions. This task creates the Zustand stores and MMKV persistence layer that the entire app depends on: settings, mastery data, and current round state. The quiz engine (Task 005), screens, and stats dashboard all read/write through these stores.

## Steps
1. Set up MMKV storage instance for persistent data.
2. Create `stores/settingsStore.ts`: Zustand store for AppSettings with MMKV persistence. Include sensible defaults (roundLength: 10, fretRange: 0–12, smartWeighting: true, etc.).
3. Create `stores/masteryStore.ts`: Zustand store for all 72 ComboMastery records with MMKV persistence. Include actions to initialize all combos as 'new', update a combo after an attempt (using scoring utils), and query weak/strong combos.
4. Create `stores/roundStore.ts`: Zustand store for current round state (not persisted — ephemeral). Track current question index, questions list, results so far, and round status (idle/active/finished).
5. Create `hooks/useMasteryData.ts`: convenience hook wrapping the mastery store with derived data — weakest 5 combos, recently improved, overall stats aggregation.

## Acceptance Criteria
- [ ] Settings store persists across app restarts via MMKV
- [ ] Mastery store initializes 72 combo records and persists updates
- [ ] Round store manages quiz session state (start, record answer, finish)
- [ ] useMasteryData hook returns weakest combos, overall accuracy, and total rounds
- [ ] All stores are importable and functional
