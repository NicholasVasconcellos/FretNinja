# Task 001: Project Scaffolding & Foundation

## Context
This is the first task. Nothing exists yet. Set up the Expo project with prebuild workflow, Expo Router, theme system, shared types, and the root/tab navigation structure. Everything else builds on this.

## Steps
1. Initialize an Expo project with the prebuild (bare) workflow and TypeScript template.
2. Install core dependencies: expo-router, react-native-mmkv, zustand, nativewind (or set up StyleSheet-based theming).
3. Set up the root layout (`app/_layout.tsx`) with font loading and any top-level providers.
4. Set up the tab navigator layout (`app/(tabs)/_layout.tsx`) with three tabs: Home, Stats, Settings. Use neon-on-dark styling for the tab bar.
5. Create placeholder screens for all routes: `app/(tabs)/index.tsx`, `app/(tabs)/stats.tsx`, `app/(tabs)/settings.tsx`, `app/quiz.tsx`, `app/results.tsx`.
6. Define the theme constants (`constants/theme.ts`): color palette (dark backgrounds, neon green #39FF14, electric blue, hot pink/red), typography scale, spacing.
7. Define the fretboard constants (`constants/fretboard.ts`): standard tuning open notes, chromatic note list.
8. Define all shared TypeScript types (`types/index.ts`): Note, GuitarString, NoteCombo, QuestionResult, RoundResult, ComboMastery, AppSettings.

## Acceptance Criteria
- [ ] `npx expo start` launches without errors
- [ ] Tab navigation works between Home, Stats, and Settings
- [ ] Stack navigation can push to Quiz and Results screens
- [ ] All types are exported from `types/index.ts`
- [ ] Theme constants are importable and contain the dark/neon color palette
- [ ] Fretboard constants define standard tuning
