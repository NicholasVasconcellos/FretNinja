# Task 013: Audit Current Pitch Detection Implementation

## Context
All 12 original tasks are complete. The app currently uses `react-native-pitch-detector` (v0.1.6, wraps Beethoven on iOS) for pitch detection. Before building a custom native module to replace it, we need to audit every touchpoint of the current implementation. This creates a migration map that task 021 will use for a clean swap.

## Steps
1. Catalog all files that import from or reference `react-native-pitch-detector`:
   - `hooks/usePitchDetection.ts` — the main hook wrapping the library
   - `app/quiz.tsx` — consumes the hook via `usePitchDetection()`
2. Catalog all supporting files:
   - `utils/notes.ts` — `frequencyToNote()` and `centsOffPitch()` (frequency-to-note conversion, will be superseded by C++ note mapper)
   - `components/PitchIndicator.tsx` — UI consumer displaying detected note/frequency
   - `plugins/withMicrophonePermission.js` — custom Expo config plugin injecting `react-native-permissions` setup into Podfile
   - `utils/sounds.ts` — audio session config (`setAudioModeAsync({ allowsRecording: true })`)
3. Catalog the dependency chain:
   - `react-native-pitch-detector` in `package.json`
   - `react-native-permissions` (transitive dependency)
   - `patches/react-native-pitch-detector+0.1.6.patch` (patch-package patch)
   - Beethoven pod (iOS, pulled in by `react-native-pitch-detector`)
4. Document the interface contract of `usePitchDetection`:
   - Returns `{ detectedNote: Note | null, frequency: number | null, isListening: boolean, start(): Promise<void>, stop(): Promise<void> }`
   - Filtering: `MAX_CENTS_DEVIATION = 40`, `MIN_FREQUENCY = 60`, `MAX_FREQUENCY = 1400`
   - App state handling: pauses on background, resumes on foreground
5. Document audio session interaction:
   - `utils/sounds.ts` calls `setAudioModeAsync({ allowsRecording: true })` to set AVAudioSession to `.playAndRecord`
   - `loadSounds()` must be called before `pitch.start()` (see commit `5d6a9f9`)
   - The new module's AVAudioEngine must coexist with this session config
6. Create `tasks/migration-map.md` documenting all findings from steps 1–5.
7. Verify the app builds and runs correctly as a baseline:
   - Run `npx expo prebuild` for both platforms
   - Verify build succeeds (iOS: `xcodebuild`, Android: `./gradlew assembleDebug`)

## Acceptance Criteria
- [ ] `tasks/migration-map.md` exists with a complete inventory of files, dependencies, config entries, and interface contracts
- [ ] The interface contract of `usePitchDetection` is documented precisely (return type, filtering constants, lifecycle behavior)
- [ ] Audio session interaction between `expo-audio` and pitch detector is documented
- [ ] The dependency chain (`react-native-pitch-detector` → `react-native-permissions` → Beethoven pod) is documented
- [ ] The patch file is identified and documented
- [ ] `npx expo prebuild` succeeds for both platforms
- [ ] Build succeeds on at least one platform (iOS or Android)
