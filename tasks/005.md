# Task 005: Quiz Engine

## Context
Tasks 002-004 provide the data layer, stores, and pitch detection. This task builds the core game logic: the quiz engine hook that orchestrates prompt generation, answer checking against detected pitch, scoring, timer management, and round flow. The quiz screen UI (Task 007) will consume this hook.

## Steps
1. Implement `hooks/useQuizEngine.ts` as the central quiz state machine. It should:
   - Generate a round of prompts (N questions based on settings) using either random or smart-weighted selection.
   - Expose the current prompt (note + string), question index, and round progress.
   - Accept detected notes from usePitchDetection and check if they match the prompted note.
   - Track response time per question (from prompt display to correct detection).
   - Handle the timer: count down per question if enabled, auto-fail on timeout.
   - Auto-advance to the next prompt after a correct answer (with configurable delay).
   - On round completion, compute the RoundResult, update mastery store, and signal navigation to results.
2. Wire the engine to read settings from the settings store (round length, fret range, timer, smart weighting).
3. Ensure the engine is decoupled from UI — it exposes state and actions, not rendering logic.

## Acceptance Criteria
- [ ] Engine generates the correct number of prompts based on round length setting
- [ ] Smart weighting is applied when enabled in settings
- [ ] Fret range setting constrains which combos appear
- [ ] Detected note is matched against prompted note (note name only, string is honor system)
- [ ] Response time is tracked per question
- [ ] Timer counts down and auto-fails when it expires (if enabled)
- [ ] Round completes and produces a valid RoundResult
- [ ] Mastery store is updated after each round
