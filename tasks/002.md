# Task 002: Fretboard Data Layer & Utility Functions

## Context
Task 001 set up the project skeleton, types, and fretboard constants. This task builds the pure-logic utility layer: the complete fret-to-note lookup table, the smart weighting algorithm, and scoring/mastery calculation helpers. These utils are consumed by the quiz engine (Task 005) and mastery system (Task 003).

## Steps
1. Build the complete fret-to-note mapping in `utils/notes.ts`: a function or lookup table that, given a string number and fret number (0–24), returns the note name. Also export the list of all 12 notes and all 72 NoteCombo objects.
2. Implement the weighted random selection algorithm in `utils/weightedRandom.ts`: given an array of ComboMastery records, compute weights using `(1 - accuracy) + (1 / totalAttempts)`, with high base weight for unattempted combos. Return a selected NoteCombo with no immediate repeats.
3. Implement scoring and mastery helpers in `utils/scoring.ts`: functions to calculate mastery level from accuracy + response time, compute round stats (score, accuracy, avg response time), and update a ComboMastery record after a new attempt.

## Acceptance Criteria
- [ ] `getNoteAtFret(string, fret)` returns correct notes for all 6 strings across frets 0–24
- [ ] `getAllCombos()` returns exactly 72 NoteCombo objects
- [ ] Weighted random selection biases toward low-accuracy and low-attempt combos
- [ ] No immediate repeat combos from the weighted selector
- [ ] Mastery level calculation maps accuracy/speed ranges to 'new' | 'learning' | 'familiar' | 'mastered'
- [ ] Round stats calculation produces correct score, accuracy, and average response time
