# Task 021: Swap Old Pitch Detection for New Module

## Context
Tasks 014–020 built and tested the custom pitch detection module in `modules/pitch-detector/`. The old implementation using `react-native-pitch-detector` is still wired into the app. This task performs the swap: removing the old library, refactoring consumers to use the new module directly, and verifying end-to-end functionality. Refer to `tasks/migration-map.md` (created in task 013) for the complete inventory of files to change.

## Steps
1. **Refactor `app/quiz.tsx`** to import `usePitchDetector` from `modules/pitch-detector` directly:
   - Replace `usePitchDetection()` with `usePitchDetector()`.
   - Map the new interface to existing usage:
     - `pitch.detectedNote` → `pitch?.note as Note` (cast string to `Note` type from `types/index.ts`)
     - `pitch.frequency` → `pitch?.frequency ?? null`
     - `pitch.isListening` → `status === 'active'`
     - `pitch.start()` / `pitch.stop()` → pass through directly
   - Update `quiz.submitDetectedNote()` call to use the new pitch result.
2. **Refactor `components/PitchIndicator.tsx`** to accept the new module's data:
   - Update props or internal usage to work with `PitchResult` from the new module.
   - Display note name and frequency as before.
3. **Delete `hooks/usePitchDetection.ts`** — no longer needed.
4. **Update `utils/notes.ts`**:
   - The `frequencyToNote()` and `centsOffPitch()` functions are superseded by the C++ note mapper.
   - Check for other callers beyond the old hook. If only used by the old hook, remove them.
   - Preserve `getAllCombos()` and other fretboard-related exports.
5. **Remove old dependencies:**
   - `npm uninstall react-native-pitch-detector`
   - Remove `react-native-permissions` if no other consumers (verify first)
   - Delete `patches/react-native-pitch-detector+0.1.6.patch`
6. **Delete `plugins/withMicrophonePermission.js`** — the new module's config plugin handles microphone permissions.
7. **Update `app.json`**: remove `"./plugins/withMicrophonePermission"` from the plugins array. Verify the new module's plugin is already listed (added in task 014).
8. **Verify audio session coexistence**: run a quiz round and confirm `expo-audio` SFX play correctly while pitch detection is active via the new module.
9. **Delete `app/pitch-test.tsx`** — temporary test screen no longer needed.
10. **Clean build**: run `npx expo prebuild --clean` and verify both platforms build.
11. **End-to-end test**: run a full quiz round — start quiz, play notes, verify detection, verify SFX, verify results screen.
