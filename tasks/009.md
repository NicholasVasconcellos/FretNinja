# Task 009: Results Screen

## Context
After a quiz round completes (Task 007), the app navigates to the results screen with the round data. This task builds that screen: overall score, per-question breakdown, and weakest combos highlighted. The user can replay or go home from here.

## Steps
1. Build out `app/results.tsx` to receive the round result data (via route params or the round store).
2. Display the round summary at the top: score (e.g., "8/10"), accuracy percentage, average response time.
3. Show a per-question breakdown list: each question shows the prompted note+string, whether correct/wrong, the detected note (if wrong), and time taken.
4. Highlight the weakest combos from this round (the ones answered incorrectly or slowly).
5. Add "Play Again" button (navigates back to quiz with a new round) and "Home" button (navigates to home tab).
6. Style with the dark/neon theme — use green for correct, red/pink for wrong.

## Acceptance Criteria
- [x] Results screen displays correct round summary stats
- [x] Per-question breakdown shows all questions with correct/wrong status
- [x] Wrong answers show what note was detected vs. what was expected
- [x] Weakest combos from the round are visually highlighted
- [x] "Play Again" starts a new round
- [x] "Home" navigates back to the home tab
- [x] Screen follows the dark + neon theme
