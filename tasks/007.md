# Task 007: Quiz Screen UI & Components

## Context
Task 005 built the quiz engine hook. This task creates the quiz screen and all its UI components: the note prompt, timer bar, score counter, pitch indicator, and feedback overlay. This is the core gameplay screen — it needs to feel responsive and clear.

## Steps
1. Build `components/NotePrompt.tsx`: large, centered display showing "Play **[Note]** on **String [N]**". Bold, high-contrast, easy to read at a glance.
2. Build `components/TimerBar.tsx`: horizontal bar that animates down over the allotted seconds. Changes color as time runs low (green → yellow → red).
3. Build `components/ScoreCounter.tsx`: compact display showing current score (e.g., "3/10") in the top area.
4. Build `components/PitchIndicator.tsx`: small indicator showing what note the microphone is currently hearing. Helps users debug tuning issues.
5. Build `components/FeedbackOverlay.tsx`: brief overlay on correct (green flash + checkmark) and wrong (red X). Should appear and auto-dismiss quickly.
6. Wire everything together in `app/quiz.tsx`: use the quiz engine hook, display all components, handle the round flow from start to finish, and navigate to results on completion.
7. Add a mic status indicator showing that the microphone is active and listening.

## Acceptance Criteria
- [ ] Quiz screen displays the current note prompt clearly
- [ ] Timer bar animates and triggers timeout when it expires
- [ ] Score counter updates after each question
- [ ] Pitch indicator shows the currently detected note in real-time
- [ ] Correct/wrong feedback is visually distinct and auto-dismisses
- [ ] Screen auto-advances to next question after correct answer (~0.5s delay)
- [ ] Round completion navigates to results screen with round data
- [ ] Mic status is visible
