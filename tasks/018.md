# Task 018: JSI TurboModule Binding & TypeScript API

## Context
Tasks 016 and 017 implemented native audio + DSP for both platforms. The native modules expose `start()`, `stop()`, `getLatestPitch()`. This task creates the clean TypeScript TurboModule spec and the `usePitchDetector` hook. Do NOT modify `app/quiz.tsx` — that happens in task 021. Instead, create a temporary test screen for validation.

## Steps
1. Finalize the TurboModule codegen spec (`src/PitchDetectorModule.ts`):
   - `start(): Promise<void>`
   - `stop(): void`
   - `getLatestPitch(): { frequency: number, confidence: number, note: string, octave: number, cents: number }`
   - Ensure the spec matches what iOS and Android native modules return.
2. Create `src/PitchDetector.ts` — the public JS API:
   - Export `usePitchDetector(options?)` hook.
   - Options: `{ pollRate?: number (default 30), minConfidence?: number (default 0.85) }`.
   - Hook internals:
     - On `start()`: call native `start()`, then begin polling via `requestAnimationFrame` or `setInterval` at configured rate.
     - Each poll: call `getLatestPitch()` synchronously via JSI, update React state.
     - Filter results below `minConfidence` (return `null` for pitch).
     - On `stop()`: cancel polling, call native `stop()`.
     - Cleanup on unmount.
   - Return type: `{ pitch: PitchResult | null, status: PitchDetectorStatus, start, stop }`.
3. Export everything from a clean `src/index.ts` barrel file.
4. Add TypeScript types in `src/types.ts` for `PitchResult`, `PitchDetectorStatus`, `UsePitchDetectorReturn`.
5. Create a temporary test screen at `app/pitch-test.tsx` that imports `usePitchDetector`, has Start/Stop buttons, and logs live pitch data. Do NOT modify `app/quiz.tsx`.

## Acceptance Criteria
- [ ] `usePitchDetector` hook works as a standard React hook with proper lifecycle cleanup
- [ ] Polling runs at configurable FPS without blocking the JS thread
- [ ] Low-confidence readings are filtered out (return `pitch: null`)
- [ ] `status` reflects 'idle' → 'starting' → 'active' state transitions
- [ ] TypeScript types are exported and usable by the consuming app
- [ ] Console.log from `app/pitch-test.tsx` shows live pitch data on both iOS and Android
