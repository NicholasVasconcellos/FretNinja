# Task 004: Pitch Detection Integration

## Context
This is the most technically challenging piece. The app needs to listen to the guitar via the microphone and detect the played note in real-time. This task sets up the native pitch detection module and wraps it in a clean hook. The quiz engine (Task 005) consumes the hook's output to check answers.

## Steps
1. Research and install a pitch detection library compatible with Expo prebuild (e.g., `react-native-pitch-detector`, or a suitable alternative). If no library works cleanly, implement a native bridge module using platform APIs (iOS: AVAudioEngine, Android: AudioRecord) with YIN or autocorrelation.
2. Run `npx expo prebuild` to generate native projects if not already done.
3. Implement `hooks/usePitchDetection.ts`: a hook that manages microphone permissions, starts/stops listening, and exposes `{ detectedNote: Note | null, frequency: number | null, isListening: boolean, start(), stop() }`.
4. The hook should map detected frequency to the nearest note name using standard A440 tuning reference. Include a confidence threshold to filter out noise.
5. Handle edge cases: microphone permission denied, background/foreground transitions, cleanup on unmount.

## Acceptance Criteria
- [ ] Microphone permission is requested and handled gracefully
- [ ] Hook detects pitch from live microphone input and returns a note name
- [ ] Frequency-to-note mapping is accurate across the guitar range (~80Hz–1200Hz)
- [ ] Low-confidence detections are filtered out (silence/noise doesn't produce false notes)
- [ ] Hook cleans up audio resources on unmount
- [ ] Start/stop controls work reliably

## Files to Touch
- hooks/usePitchDetection.ts
- utils/notes.ts (may need frequency-to-note helper)
- app.json (microphone permission descriptions)
- ios/ and android/ (native config if using custom module)
