# Task 002: Implement C++ DSP Core (YIN + Note Mapper)

## Context
Task 001 created the module skeleton with CMake configured. This task implements the pure C++ DSP logic with no platform dependencies. It should compile and be testable in isolation.

## Steps
1. Implement `pitch_result.h` — define `PitchResult` struct: `{ float frequency, float confidence }`.
2. Implement `ring_buffer.h` — lock-free SPSC ring buffer, fixed capacity 4096 samples, float32. Methods: `write(float*, size_t)`, `read(float*, size_t)`, `available()`.
3. Implement `yin.h` / `yin.cpp` — the YIN pitch detection algorithm:
   - Constructor takes sample_rate, frame_size, threshold (defaults from plan: 44100, 2048, 0.15).
   - `PitchResult detect(const float* buffer, int length)` — runs the full YIN pipeline.
   - Internal: difference function, cumulative mean normalized difference, absolute threshold search, parabolic interpolation, frequency conversion.
   - Preallocate all working buffers in constructor — zero allocations in `detect()`.
   - Clamp output to MIN_FREQUENCY (65Hz) / MAX_FREQUENCY (4200Hz).
4. Implement `note_mapper.h` / `note_mapper.cpp`:
   - `NoteInfo mapToNote(float frequency)` returning `{ string note, int octave, float cents }`.
   - Uses MIDI formula: `midi = 12 * log2(freq / 440) + 69`, then derive note name, octave, cents.
5. Add constants to a shared header or within yin.h: SAMPLE_RATE, FRAME_SIZE, HOP_SIZE, YIN_THRESHOLD, MIN/MAX_FREQUENCY, MIN_CONFIDENCE.
6. Write a simple `main.cpp` test harness that generates sine waves at known frequencies (440Hz, 82.4Hz, 1000Hz, 4000Hz) and verifies YIN output is within ±1 cent. This file is for local validation only, not shipped.

## Acceptance Criteria
- [ ] `yin.cpp` detects A4 (440Hz) sine wave within ±1 cent accuracy
- [ ] `yin.cpp` detects E2 (82.4Hz) sine wave within ±5 cents (low freq is harder)
- [ ] `yin.cpp` returns confidence < 0.5 for silence (all zeros)
- [ ] `note_mapper` correctly maps 440Hz → A4, 0 cents
- [ ] `ring_buffer` handles wrap-around writes and reads correctly
- [ ] Zero heap allocations inside `YIN::detect()` (all buffers preallocated)
- [ ] Test harness compiles and passes locally with `g++` or `clang++`
