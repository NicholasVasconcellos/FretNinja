# Task 004: Android Audio Capture with Oboe

## Context
Tasks 001–002 are complete (scaffold + DSP). Task 003 handled iOS. This task does the same for Android: real mic input via Oboe, feeding the C++ YIN engine, results stored for JSI access.

## Steps
1. Add Oboe as a dependency in the module's `build.gradle` (use the prefab/CMake integration).
2. Create a C++ `OboeAudioCapture` class that:
   - Implements `oboe::AudioStreamCallback`.
   - Opens an input stream: float32, 44100Hz, mono, low-latency performance mode, exclusive sharing mode.
   - In `onAudioReady()`, writes samples to the shared ring buffer and triggers YIN detection when a full frame (2048 samples) is available.
   - Stores latest `PitchResult` + `NoteInfo` atomically.
3. Create JNI bindings (`pitch-detector-jni.cpp`) exposing:
   - `nativeStart()` → creates and starts the Oboe stream.
   - `nativeStop()` → stops and closes the stream.
   - `nativeGetLatestPitch()` → reads the atomic result, returns as a jobject or primitive array.
4. Wire the Kotlin TurboModule:
   - `start()` → check/request RECORD_AUDIO permission → call JNI `nativeStart()` → return Promise.
   - `stop()` → call JNI `nativeStop()`.
   - `getLatestPitch()` → call JNI `nativeGetLatestPitch()`, return as JS-compatible map.
5. Update `CMakeLists.txt` to compile all C++ files (yin.cpp, note_mapper.cpp, ring_buffer, OboeAudioCapture, JNI bindings) and link against Oboe.
6. Handle audio focus changes gracefully (other apps taking audio).

## Acceptance Criteria
- [ ] `start()` requests RECORD_AUDIO permission and begins capture on Android
- [ ] `stop()` cleanly shuts down Oboe stream
- [ ] `getLatestPitch()` returns real pitch data when a tone is played
- [ ] Oboe stream runs in low-latency / exclusive mode
- [ ] Audio callback has zero allocations, no JNI calls
- [ ] Builds via `npx expo prebuild --platform android && ./gradlew assembleDebug`
